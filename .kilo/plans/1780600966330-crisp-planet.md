# MindGate Comprehensive Fix & Windows Support Plan

## Critical Issues Identified

### 1. Missing TypeScript Component Export
**Problem**: `ChatInterface.tsx` imports `LiquidGlassButtonStyle` from `Orb.tsx` but it's not exported.
**Evidence**: `Orb.tsx` defines `FlowingLinesView` and `Orb` but no `LiquidGlassButtonStyle`.
**Impact**: TypeScript compilation will fail.

### 2. No Windows Platform Implementation
**Problem**: Electron codebase lacks Windows-specific system monitoring.
**Evidence**: No platform detection or Windows monitoring code exists in `mindgate-electron/src/platform/`.
**Impact**: App won't work on Windows.

### 3. Incomplete Electron System Monitoring
**Problem**: Electron version has no workspace monitoring - relies on Swift's `WorkspaceMonitor`.
**Evidence**: `main.ts` only has basic IPC handlers, no app monitoring logic.
**Impact**: No distraction detection in Electron version.

### 4. Overlay Not Positioned on Target Window
**Problem**: Electron's `windowManager.ts` lacks proper overlay-to-window positioning logic.
**Evidence**: `getTargetWindowFrame()` returns screen bounds instead of active window bounds.
**Impact**: Cluely-style contextual overlay won't work.

```typescript
// Platform-agnostic window monitoring
class WorkspaceMonitor {
  private getActiveWindow(): Promise<ActiveWindowInfo> {
    if (process.platform === 'win32') {
      return this.getWindowsForegroundWindow();
    } else if (process.platform === 'darwin') {
      return this.getMacActiveApp();
    }
    return this.getLinuxActiveWindow();
  }
}
```

**Electron Overlay Implementation:**
```typescript
// In windowManager.ts - overlay covers target window
showOverlay(targetFrame: Rectangle) {
  this.overlayWindow = new BrowserWindow({
    x: targetFrame.x,
    y: targetFrame.y,
    width: targetFrame.width,
    height: targetFrame.height,
    transparent: true,
    frame: false,
    alwaysOnTop: true,
    focusable: false,
    skipTaskbar: true
  });
  this.overlayWindow.setIgnoreMouseEvents(true);
  // Dark semi-transparent backdrop
  // Orb appears anchored to this overlay
}
```

**Key for Cluely UX:**
- Overlay appears only on the window with distracting content
- Orb positions relative to that window (e.g., top-right corner)
- No full-screen takeover - just contextual interruption
- Smooth show/hide animations with Framer Motion
- Overlay is click-through, only orb is interactive

## Detailed Implementation Specifications

### Windows Foreground App Detection

**PowerShell Script for Window Info:**
```typescript
// GetForegroundWindow + GetWindowRect via PowerShell
const getWindowsWindowInfo = async (): Promise<ActiveWindowInfo> => {
  const psScript = `
Add-Type @"
using System;
using System.Diagnostics;
using System.Runtime.InteropServices;
public class Win32 {
  [DllImport("user32.dll")]
  public static extern IntPtr GetForegroundWindow();
  [DllImport("user32.dll")]
  public static extern bool GetWindowRect(IntPtr hWnd, out RECT lpRect);
  [DllImport("user32.dll")]
  public static extern uint GetWindowThreadProcessId(IntPtr hWnd, out uint lpdwProcessId);
}
public struct RECT {
  public int Left, Top, Right, Bottom;
}
"@
$hwnd = [Win32]::GetForegroundWindow()
$rect = New-Object RECT
[Win32]::GetWindowRect($hwnd, [ref]$rect)
$processId = 0
[Win32]::GetWindowThreadProcessId($hwnd, [ref]$processId)
$process = Get-Process -Id $processId -ErrorAction SilentlyContinue
if ($process) {
  "$($process.ProcessName)|$($process.MainWindowTitle)|$($rect.Left)|$($rect.Top)|$($rect.Right-$rect.Left)|$($rect.Bottom-$rect.Top)"
}
`;
};
```

### Window Positioning for Cluely-like Overlay (Windows/macOS/Linux)

**Overlay Strategy:**
- When distraction detected, get target window frame
- Create overlay BrowserWindow matching target window bounds  
- Apply semi-transparent backdrop (e.g., rgba(0,0,0,0.3))
- Position orb at top-right corner of overlay + 12px offset
- On Windows: Use PowerShell to get window bounds
- On macOS: Use AppleScript or Accessibility API
- On Linux: Use X11/Wayland window listing

**Window Level Handling:**
- Overlay: Level above target app but below always-on-top
- Orb: Always on top (screenSaver level)
- Click-through for overlay (ignoreMouseEvents: true)
