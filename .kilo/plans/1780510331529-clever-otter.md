# MindGate Launch-Readiness Plan

## Critical Issues to Fix

### 1. Info.plist Configuration
**Problem:** The Info.plist references `$(PRODUCT_BUNDLE_IDENTIFIER)` which is not defined in Package.swift. The activation policy conflicts with LSUIElement setting.

**Fixes:**
- Add package product bundle identifier to Package.swift
- Add proper entitlement strings for sandboxing
- Resolve activation policy conflict (LSUIElement=yes means accessory, but main.swift uses .regular)

### 2. Activation Policy Fix in main.swift
**Problem:** main.swift sets `app.setActivationPolicy(.regular)` but Info.plist has `LSUIElement = true` which typically should use `.accessory`.

**Fix:** Change to `.accessory` for proper background agent behavior without Dock icon.

### 3. Configuration Not Applied
**Problem:** ChatView has hardcoded `countdownSeconds = 20` but Configuration has `justificationCountdownDuration` that should be used.

**Fix:** Use `configuration.settings.justificationCountdownDuration` instead of hardcoded value.

### 4. Ollama URL Connection Check
**Problem:** The `checkConnection()` method in OllamaService uses naive URL replacement that fails with paths not ending in `/api/generate`.

**Fix:** Use proper URL path manipulation with `URL.appendingPathComponent()` and handle edge cases.

### 5. Missing Error Handling for AppleScript
**Problem:** `closeBrowserTab()` in DecisionEngine doesn't handle all error cases gracefully and uses force-unwrapped strings.

**Fixes:**
- Add proper error logging and user-facing feedback
- Handle nil bundle identifiers gracefully
- Add fallback for browsers that don't support AppleScript scripting

### 6. Race Conditions in Workspace Monitoring
**Problem:** Both NSWorkspace notifications and polling run simultaneously without synchronization. `lastCheckedApp` comparison may miss edge cases.

**Fixes:**
- Add serial queue for state mutations
- Prevent concurrent evaluation of same app
- Add hysteresis to prevent rapid show/hide cycles

### 7. Memory Management Issues
**Problem:** Timer references in WindowManager and ChatView may not be properly cleaned up, leading to retain cycles.

**Fixes:**
- Use `[weak self]` consistently in all timer closures
- Add explicit cleanup in `deinit` methods
- Cancel timers before view dismissal

### 8. SettingsView Incomplete Implementation
**Problem:** Access durations, productive tasks, and productive apps show as read-only Text but have "TODO" comments indicating they should be editable.

**Fixes:**
- Implement editing UI for access durations
- Add editable list for productive tasks
- Make productive apps list editable

### 9. Missing Validation in Settings
**Problem:** No input validation for URLs, model names, or app names in SettingsView.

**Fixes:**
- Validate Ollama URL format
- Validate model name is not empty
- Sanitize app/keyword input

### 10. Missing Ollama Connection Status
**Problem:** No status indicator in UI for Ollama connectivity.

**Fix:** Add connection status check and display in status bar menu or Orb view.

### 11. StatusBar Menu Not Reflecting App State
**Problem:** Menu shows static items but doesn't update when settings change or connection status changes.

**Fixes:**
- Add dynamic menu items that reflect current state
- Add indicator for when distraction prompt is active

## Feature Enhancements

### 12. Launch Agent Support
**Add:** Create a launch agent plist to allow auto-start on user login.

### 13. Logging to File
**Add:** Write logs to file in Application Support directory for debugging.

### 14. Crash Protection
**Add:** Wrap critical operations in do-catch blocks to prevent app crashes.

### 15. Window Level Management
**Fix:** Ensure Orb stays visible during full-screen apps and across all spaces properly.

### 16. Unused TakeoverView
**Problem:** TakeoverView exists but is never actually presented (referenced in ChatView state but never triggered).

**Fix:** Either implement the takeover flow or remove dead code.

### 17. Redundant AccessibilityManager
**Problem:** WorkspaceMonitor creates its own AccessibilityManager (line 29) even though AppDelegate already has one.

**Fix:** Remove the redundant instantiation and use the injected reference.
