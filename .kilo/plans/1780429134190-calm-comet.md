# Plan: Fix typing in ChatView + commit/push

## Problem
When MindGate shows the orb/prompt for a distracting app, the user cannot type in the justification text field.

## Root cause
`ChatView` uses `FocusableTextField` (an `NSTextField` wrapper). In a borderless, `hidesOnDeactivate = false` panel that is brought front with `orderFrontRegardless`, `NSTextField` often does not reliably become first responder, so keyboard input is lost.

## Fix
Replace `FocusableTextField` with the more reliable `ReliablePromptTextView` (`NSTextView` via `NSViewRepresentable`) that is already implemented in `ChatView.swift`. `NSTextView` handles first responder and editing in this context much better.

## Changes
1. `Sources/MindGate/Views/ChatView.swift`:
   - Switch `promptBox` to use `ReliablePromptTextView` instead of `FocusableTextField`.
   - Remove `FocusableTextField` / `CustomTextField` since they’re no longer needed.

2. Build verify.
